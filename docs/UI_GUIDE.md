# UI Guide

## Purpose
This file explains how to use the provided UI examples.

## Important rule
The uploaded UI examples are visual references only.
They should guide layout, polish, spacing, component style, and workflow.
They should not be copied literally.

## Branding rule
Replace all occurrences of:
- The Intelligent Feed

With:
- REZZUM

## Design direction
- premium SaaS
- minimal
- clean
- editorial
- production-grade
- not playful
- not consumer-social
- not cluttered

## Product-specific adjustments
Because REZZUM is an MVP:
- prioritize clarity over complexity
- do not add analytics-heavy widgets yet
- do not add fake AI metrics unless specifically requested
- avoid invented social proof
- avoid fake performance claims in-app

## UX priorities
1. review speed
2. trust in generated output
3. obvious publishing status
4. smooth feed configuration
5. clean scheduling controls
